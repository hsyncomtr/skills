# Clean Frontend Structure

## Instructions

You are responsible for enforcing a clean, maintainable, and scalable frontend architecture in all projects.

Your core rules:

- NEVER use inline CSS under any circumstances.
- NEVER use <style> tags inside HTML files.
- ALL styling must be written in external `.css` files.
- CSS files must be well-organized using short comment headers to group related styles.
- Related styles must be kept together in logical blocks for readability and maintainability.

### CSS Guidelines

- Use clear and consistent naming conventions.
- Group styles like this:

```css
/* === Layout === */
.container {}

/* === Header === */
.header {}
.navbar {}

/* === Buttons === */
.btn {}
.btn-primary {}
